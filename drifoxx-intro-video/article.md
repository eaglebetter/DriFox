# DriFoxx 核心功能介绍

## 一、Hooks 系统 —— 你的 AI 工作流守门人

DriFoxx 的 Hooks 系统是整个工具链的"事件拦截器"。它的工作原理很直接：在 AI 对话流程的关键节点，你可以插入自定义逻辑，决定这个操作是继续、阻止、还是变形。

七个事件钩子覆盖了完整的对话生命周期：SessionStart（会话开始）、PreUserMessage（用户消息发送前）、PostUserMessage（用户消息发送后）、PreAssistantMessage（AI 回复前）、PostAssistantMessage（AI 回复后）、PreToolUse（工具调用前）、PostToolUse（工具调用后）。

每种 Hook 支持三种类型：command（执行 shell 命令）、http（发送 HTTP 请求）、python（调用 Python 函数）。command 类型最常用，比如在调用 webfetch 之前，你可以用一个 `exit 2` 的 command hook 直接拦截掉，防止 AI 随意抓取网页。http 类型适合通知外部系统，比如每次 AI 调用工具后给你的企业微信发一条消息。python 类型最灵活，可以直接调用项目内的函数做复杂逻辑。

Matcher 机制让你精确控制 Hook 的触发条件。`tool:webfetch` 只在调用 webfetch 工具时触发，正则表达式 `.*帮助.*` 只在用户消息包含"帮助"时触发。没有 matcher 则对事件下所有操作生效。

Hook 的决策能力是关键：返回 `continue` 放行操作，返回 `block` 直接拦截，返回 `defer` 延迟执行。这让 Hook 不只是通知器，而是真正的流程控制器。

技能（Skill）也可以注册自己的 Hook。技能 Hook 在 UI 上标记为只读，用户不能删除，但可以关闭。全局 Hook 和技能 Hook 在同一个事件分组里展示，一目了然。

配置存储在 `.drifox/hooks/hooks.json`，UI 上支持添加、编辑、启用/禁用、删除，修改后热重载，不用重启。

## 二、MCP 系统 —— 让 AI 连接一切工具

MCP 是 Model Context Protocol 的缩写，DriFoxx 通过它把外部工具服务器接入 AI 的能力范围。简单说，你的 AI 突然能用浏览器操控文件系统、查数据库、调 API —— 这些都是 MCP 服务器提供的。

连接方式有三种：stdio（标准输入输出，适合本地命令行启动的 MCP 服务）、sse（Server-Sent Events，适合远程 HTTP 服务）、http（Streamable HTTP，最新的 MCP 传输协议）。配置一个 MCP 服务器只需要填名称、类型、启动命令或 URL，点击连接就行。

MCPClientManager 是全局单例，所有窗口共享同一个连接池。每个服务器连接由一个持久 asyncio Task 管理，在专属后台线程的事件循环中运行，不阻塞 UI。连接建立后自动发现工具列表，每个工具自动加上 `mcp__服务器名__工具名` 的前缀，注入到 AI 的 function calling schema 里。AI 看到这些工具描述，就像看到内置工具一样自然调用。

DriFoxx 还支持自动发现已有配置。Claude Desktop、Cursor、Windsurf、VS Code 这些主流工具的 MCP 配置，DriFoxx 启动时自动扫描并导入，不用重复配置。连接支持热添加和热断开，不用重启应用。

调用工具时，参数通过 MCP 协议传到对应服务器，结果原路返回。整个过程对用户透明 —— 你只看到 AI 在用工具，背后是 MCP 在运转。

## 三、AutoLoop 系统 —— 全自动任务执行引擎

AutoLoop 是 DriFoxx 最强的自动化能力：给一个任务描述，它自己规划、自己执行、自己验证，循环往复直到完成。

设计上分两个阶段。第一阶段是规划（Planning）：AI 只拆解任务，不写代码。它把任务拆成带编号的步骤，每步包含描述、涉及文件、验证方式、需求验证点，全部写入 SHARED_TASK_NOTES.md 这个共享文档。规划完成的标准是输出 PLANNING_COMPLETE 且笔记里有有效步骤。

第二阶段是执行（Executing）：一轮一步，增量推进。每步执行完后验证结果，通过 [x] 勾选完成的步骤，前进到下一步。如果验证连续失败 3 次，自动跳过避免卡住。每轮结束，AI 必须在 SHARED_TASK_NOTES.md 末尾追加操作记录，禁止覆盖原始计划。

安全控制有三道防线：迭代轮数上限（默认 50 轮）、Token 用量上限（默认 50 万）、运行时长上限（默认 2 小时）。任何一道超限，自动停止。完成信号检测需要连续出现指定次数的"DONE"关键词（默认 3 次），避免误判。

UI 上，AutoLoopConfigCard 提供完整配置：最大轮数、Token 上限、时长限制、完成信号词、确认次数、项目路径。AutoLoopRunningCard 用彩虹渐变动画边框实时显示进度，包括当前步骤、已验证步骤、迭代数、耗时、Token 消耗。

每轮执行的完整日志独立保存到 `.autoloop/logs/round_001.md`，方便事后审计。

## 四、项目管理系统 —— 让 AI 记住你的项目上下文

DriFoxx 的项目管理系统解决一个核心问题：AI 需要知道你在做什么项目、项目里有什么重要文件、你积累过什么经验。三个子系统各司其职。

项目笔记（Project Notes）是一个 Markdown 编辑器，每个项目独立维护。AI 在对话中会自动读写项目笔记 —— `edit_project_note` 工具让它用精确字符串替换更新笔记，`read_project_note` 工具让它读取当前笔记内容。你做的设计决策、重要的约束条件、踩过的坑，AI 都会记下来，下次对话直接参考。

关键文档（Key Documents）是一组文件路径，告诉 AI 这个项目里哪些文件最重要。添加方式很直觉：直接拖拽文件到关键文档列表，或者在聊天中用 `stage_files` 工具标记。关键文档的路径会注入到 AI 的上下文里，让它在回答时优先参考这些文件。文件夹也可以作为关键文档添加，此时该文件夹会作为工具的工作目录。

项目根目录（Project Root）是最基础但最重要的设置。它决定了相对路径的基准、文件搜索的范围、AutoLoop 的工作目录。在设置里切换项目根目录，AI 的上下文就跟着切换。

三个子系统通过 MemoryCard 统一管理，三个标签页切换：条目记忆（零散的关键信息）、项目笔记（结构化的项目知识）、关键文档（重要文件路径）。所有数据持久化在 SQLite 数据库里，关了应用不会丢。

## 五、主题与字体 —— 让界面属于你

DriFoxx 提供四套暗色主题，每套从窗口背景到输入框边框都有完整的色彩体系，不是简单的换色。

深海蓝黑（Midnight）：冷调蓝黑底色配冰蓝强调色，适合长时间使用，眼睛最不累。曜石紫（Obsidian）：深紫底色配亮紫强调色，视觉上最有质感。松林暗绿（Forest）：暗绿底色配翡翠绿强调色，自然感强，写代码时心态最稳。石墨铜（Graphite）：纯灰底色配铜金强调色，最接近编辑器风格，适合重度码字。

字体大小三档可选：小（13px 基准）、中（14px 基准，默认）、大（16px 基准）。每档不只是改基准字号，整个 UI 的间距、行高、内边距都按比例联动调整，保证视觉比例不崩。

主题切换实时生效，所有卡片、输入框、按钮、文字颜色跟着变，不需要重启。Design Tokens 统一管理所有颜色变量，开发者新增组件只要引用 token 就自动适配四套主题。
