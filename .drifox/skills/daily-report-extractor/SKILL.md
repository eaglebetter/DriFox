# 日报提取与绩效生成技能

## 工作流程

### 第一步：提取图片文字

使用 MiniMax API 分析图片：

```bash
cd "D:/work/DriFox/app/skills/minimax-image-understanding"
set MINIMAX_API_KEY=sk-cp-YECP4gj2VZt7NxgUYww5Shn5FXBqb8aSIQDD0zMUoLzx10y4KEAYFKHqrmGhmx5Cxt9KV030zSXvyhDyqVs_0n-R2djEgYx6fzMs46T6gf3HooJdS5c4_Kk
python scripts/analyze_image.py --file "{图片路径}" --prompt "请逐行完整提取图片中所有日报/工作记录的文字内容，按照原文格式一行一行输出，不要遗漏任何字符，包括标点符号"
```

### 第二步：确定月份和周划分

根据日报内容中的日期确定月份，然后根据国务院节假日安排确定工作日：

**2026年节假日安排参考：**
- 元旦：1月1日至3日放假，共3天
- 春节：2月15日至23日放假，共9天（2月14日、28日上班）
- 清明节：4月4日至6日放假，共3天
- 劳动节：5月1日至5日放假，共5天（5月9日上班）
- 端午节：6月19日至21日放假，共3天
- 中秋节：9月25日至27日放假，共3天
- 国庆节：10月1日至7日放假，共7天（9月20日、10月10日上班）

**周划分格式示例：**
```
第一周（2026/4/1至3）
第二周（2026/4/6至10）
...
```

### 第三步：整理日报格式

将提取的文字按日期分行，每行格式：
```
日期 今日完成任务：[任务描述] 明日任务：[任务描述]
```

### 第四步：生成月度绩效报告

#### 周工作总结格式：
```
第一周（YYYY/M/D至M）工作总结
【项目名称】
1. 完成的任务描述1。
2. 完成的任务描述2。

【项目名称】
1. 完成的任务描述。
```

#### 月度总结格式：
```
月度总结

1. 产品研发
【大模型项目】
1. 完成的任务描述。

【视觉模型项目】
1. 完成的任务描述。

2. 项目支持
【项目名称】
1. 完成的任务描述。

3. 材料编写、培训、分享
【培训与材料】
1. 完成的任务描述。
```

## 输出要求

1. **不提进度百分比**：绩效中只写完成了哪些内容，不写"进度：XX%"
2. **不加序号点分项**：项目标题用【】包裹，每项内容用数字编号
3. **不写"今日完成任务"字样**：直接写任务内容
4. **按周分节**：每周一个章节，最后是月度总结

## 注意事项

- MiniMax API Key 已配置在环境变量中
- 图片路径支持绝对路径和相对路径
- 如果用户没有指定输出文件路径，可以直接输出到对话中
