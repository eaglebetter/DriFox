# 布局规则

## 坐标系统

- 原点：画布左上角 (0, 0)
- x 轴：向右递增
- y 轴：向下递增
- 单位：像素 (px)

---

## 默认尺寸

| 节点类型 | 宽度 | 高度 |
|---------|------|------|
| 标准节点 (llm/agent/knowledge等) | 180px | 60px |
| start / end | 72px | 72px |
| 容器 (container) | 320px | 180px |

---

## 间距规则

### 水平间距
- **节点间距**：240px（从左边缘到下一个节点左边缘）
- **起始位置**：x = 80
- **计算公式**：第 N 个节点 x = 80 + (N-1) × 240

### 垂直间距
- **并行节点间距**：80px
- **分支节点偏移**：相对主节点 y ± 80

---

## 多流程排列

当一个画布包含多个独立流程时，按垂直方向排列：

```
流程 1: 主轴 y = 240
流程 2: 主轴 y = 480
流程 3: 主轴 y = 700
```

---

## 分支布局

### 分类器分支（横向分支）
```
          → 分支A (x+240, y-80)
start → classifier → 分支B (x+240, y)
          → 分支C (x+240, y+80)
```

### 知识库辅助（上方辅助）
```
      知识检索 (x, y-100)
            ↓
start → 主节点 (x, y) → ...
```

---

## 容器内布局

容器内的子节点相对于容器定位：

```
容器: x=1000, y=200, 320×180

子节点应放在:
- 左上: x = 容器.x + 80 = 1080
- 上中: y = 容器.y + 60 = 260

确保子节点中心在容器矩形内
```

---

## 坐标计算示例

### 简单流水线（5个节点）
```
n1 start:     x=80,  y=240
n2 llm:       x=320, y=225 (居中偏移 -15)
n3 agent:     x=560, y=225
n4 llm:       x=800, y=225
n5 end:       x=1040,y=240
```

### 带分类器分支（3个分支）
```
n1 start:        x=80,  y=240
n2 classifier:   x=320, y=240
n3a 分支A:       x=560, y=160
n3b 分支B:       x=560, y=240
n3c 分支C:       x=560, y=320
n4 reply:        x=800, y=240
n5 end:          x=1040,y=240
```

---

## 边界约束

- **最小 y 值**：80（防止节点超出顶部）
- **最小 x 值**：80
- **容器内子节点**：必须完全在容器边界内

---

## 对齐技巧

1. **水平对齐**：同类型节点使用相同的 y 基准值
2. **垂直居中**：非 start/end 节点 y = 主轴 - 15（60-72）/2
3. **分支等距**：多个分支使用相同的 x 增量
4. **容器居中**：子节点整体在容器内水平居中
